# Docs Visuals

This folder contains source-controlled visuals used by the project docs.

The goal is to treat diagrams like code: the editable source lives in the repo, the rendered asset is generated by a script, and both can be reviewed in pull requests.

## Structure

```text
docs/visuals/
  source/      # Editable HTML/CSS/SVG sources
  rendered/    # Generated PNG files
```

Use `source/` for the canonical visual source. Use `rendered/` for committed output files referenced by README files or other Markdown docs.

## Rendering

Regenerate all visuals from the repo root:

```sh
pnpm docs:visuals
```

The renderer opens each `docs/visuals/source/*.html` file with Playwright and screenshots the element marked with `data-visual-root`.

## Authoring Guidelines

- Use HTML as the rendering container, not as the visual language by default.
- Prefer SVG inside the HTML for diagrams that need precise positioning.
- Keep prose in Markdown when possible. The rendered image should mostly contain diagram labels, not explanatory paragraphs.
- Commit both the source HTML and rendered PNG if the PNG is referenced in repo docs.
- Use transparent PNG backgrounds unless the visual requires a fixed backdrop.

## Markdown Usage

Reference rendered images with relative paths:

```md
![Diagram comparing native VM networking with browser VM networking](docs/visuals/rendered/browser-network-stack.png)
```
